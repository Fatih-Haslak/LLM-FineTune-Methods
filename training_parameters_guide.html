Donanım & Quantization
9B parametreli modeli 16 GB VRAM'e sığdırma teknikleri
LOAD_4BIT
CONFIG → BitsAndBytesConfig
True
Modeli 4-bit NF4 formatında yükler. 9B parametreli Gemma modeli normalde fp16'da ~18 GB VRAM ister. 4-bit ile bu ~4.5 GB'a iner.
fp16 (LOAD_4BIT=False)~18 GB
4-bit NF4 (LOAD_4BIT=True) ✓~4.5 GB
4-bit + double_quant + LoRA~6 GB
✓ 16 GB GPU için True olmalı. False yaparsan model yüklenemez.
VRAM KritikZorunlu
USE_BF16
CONFIG → fp16/bf16
False
Hesaplama sırasında kullanılan floating-point formatını seçer. FP16 ve BF16 aynı bellek boyutunu kullanır ama davranışları farklıdır.
FP16: sign(1) | exponent(5) | mantissa(10) → küçük overflow riski var (NaN oluşabilir) → RTX 3090 / 4090 için önerilir BF16: sign(1) | exponent(8) | mantissa(7) → overflow riski yok (FP32 ile aynı aralık) → A100 / H100 için önerilir
RTX GPU'larda BF16 donanım desteği yok → False bırak. A100/H100'de True yap.
StabiliteHız
bnb_4bit_quant_type
BitsAndBytesConfig
"nf4"
4-bit quantization formatı. NF4 (NormalFloat4), sinir ağı ağırlıklarının normal dağılımını baz alarak optimize edilmiş, QLoRA makalesinin önerdiği format.
Normal ağırlık dağılımı (yaklaşık): %68'i → [-1σ, +1σ] arasında %95'i → [-2σ, +2σ] arasında NF4 → bu dağılıma göre 16 seviye ayarlar → her seviye eşit sayıda ağırlık içerir → en verimli 4-bit temsil fp4 → sabit aralıklı 16 seviye → daha az doğru
DoğrulukBellek
bnb_4bit_use_double_quant
BitsAndBytesConfig
True
İki aşamalı quantization. Ağırlıkları quantize ederken kullanılan "quantization sabitleri" de 8-bit'e indirilir.
Normal quantization: W_4bit + quantization_constants (fp32) → sabitler W'nin ~%25'i kadar yer tutar Double quantization: W_4bit + quantized_constants (8bit) → 9B model için ek ~0.4 GB tasarruf
Neredeyse sıfır doğruluk kaybıyla ~0.4 GB VRAM kazanımı → her zaman True bırak.

512
Token cinsinden maksimum dizi uzunluğu. Bu değerden uzun örnekler sağdan kırpılır. Kısa örnekler sağdan padding ile doldurulur.
Bir eğitim örneği: [system_msg][user_msg][context][soru][cevap] ← ─ ─ ─ ─ ─ ─ MAX_SEQ_LEN = 512 token ─ ─ ─ → 512 token içindeyse: tüm örnek kullanılır ✓ 512 tokeni geçiyorsa: sağdan kırpılır ⚠ (cevabın sonu kaybolabilir!) VRAM kullanımı: 512 → ~6 GB (güvenli 16GB'da) 1024 → ~9 GB (sınırda) 2048 → ~16 GB (OOM riski)
Hukuki metinler uzun olabilir. Kırpma olup olmadığını log'daki "Truncated?" satırından kontrol et.

Batch & Gradient Ayarları
Bellek kısıtları altında verimli eğitim
BATCH_SIZE
SFTConfig → per_device_train_batch_size
1
Her GPU'nun her adımda kaç örnek işleyeceği. 9B model + 4-bit + gradient checkpointing ile 16 GB'da güvenli maksimum 1 veya 2'dir.
Artırmadan önce VRAM kullanımını izle. OOM hatası alırsan hemen 1'e düşür.
Batch'i büyütmek için BATCH_SIZE değil GRAD_ACCUM artır — sonuç aynı, bellek kullanımı farklı.
VRAM KritikOOM Riski
GRAD_ACCUM
SFTConfig → gradient_accumulation_steps
16
Gradient Accumulation — N adımda gradyanları biriktirir, sonra tek bir optimizer güncellemesi yapar. Belleği artırmadan büyük batch etkisi sağlar.
Efektif Batch = BATCH_SIZE × GRAD_ACCUM = 1 × 16 = 16
Adım 1: örnek → loss → grad biriktirilir Adım 2: örnek → loss → grad biriktirilir ... Adım 16: örnek → loss → optimizer.step() ✓ = 1 büyük batch (16 örnek) etkisi = ama sadece 1 örneğin VRAM'i kullanıldı

True
Forward pass sırasında ara aktivasyonları saklamaz, backward'da yeniden hesaplar. %30-40 daha az VRAM, %20 daha yavaş eğitim tradeoff'u.
Normal: forward → tüm aktivasyonlar RAM'de tutulur backward → hazır aktivasyonlar kullanılır → ~8 GB aktivasyon VRAM Gradient Checkpointing: forward → aktivasyonlar SILINIR backward → aktivasyonlar YENIDEN HESAPLANIR → ~2 GB aktivasyon VRAM → ama ~%20 daha yavaş use_reentrant=False → PyTorch 2.0+ için stabil

WARMUP_RATIO
SFTConfig → warmup_ratio
0.05
Toplam adımların ilk %5'inde LR, 0'dan peak değere lineer olarak yükselir. İlk adımlardaki büyük, kararsız gradyanlardan modeli korur.
Örnek: 1000 toplam adım Adım 1–50: LR = 0 → 2e-4 (warmup) Adım 51–950: LR = 2e-4 → 0 (cosine decay) Warmup neden gerekli? İlk adımlarda ağırlıklar rastgele → gradyanlar çok büyük ve gürültülü → yüksek LR ile diverge edebilir → warmup ile yumuşak başlangıç ✓
Diverge Önleme
MAX_GRAD_NORM
SFTConfig → max_grad_norm
1.0
Gradient Clipping — tüm gradient vektörünün L2 normu bu değeri geçerse orantılı olarak küçültülür. Exploding gradient'i önler.
Her optimizer adımında: ‖g‖ = sqrt(Σ gᵢ²) ← L2 norm hesapla ‖g‖ = 0.7 → değişmez (0.7 < 1.0) ‖g‖ = 1.0 → değişmez (eşit) ‖g‖ = 3.5 → g = g × (1.0/3.5) (kırpıldı) ‖g‖ = 8.0 → g = g × (1.0/8.0) (kırpıldı) Kırpma olmadan: loss aniden ∞ olabilir (NaN)

weight_decay
SFTConfig → weight_decay
0.01
L2 Regularization. Her adımda ağırlıkları hafifçe sıfıra doğru iter. Büyüyen ağırlıkları cezalandırarak overfitting'i azaltır.
Loss_total = cross_entropy(y, ŷ) + 0.01 × Σ(wᵢ²) Güncelleme: w ← w × (1 - lr × 0.01) - lr × ∇L

optim
SFTConfig → optim
"paged_adamw_8bit"
Paged AdamW 8-bit — BitsAndBytes'ın hafıza optimizasyonlu optimizer'ı. İki özellik sunar: optimizer durumları 8-bit'e indirgenir ve bellek taşarsa CPU'ya sayfalanır.
